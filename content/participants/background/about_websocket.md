## WebSocketって何？

**WebSocket** は、サーバとクライアントの間で **双方向に・つなぎっぱなしで** メッセージをやり取りするための通信方式です。
ウェブで一般的な HTTP が「リクエストを送ったらレスポンスが返ってきて終わり」という1往復型なのに対し、WebSocket は **一度つながったら、そのまま好きなタイミングでメッセージを送り合える** のが特徴です。

チャットアプリやオンラインゲームのように、サーバから「いまイベントが起きたよ」とリアルタイムに通知したい場面でよく使われます。

---

## HTTPとの違い

| | HTTP | WebSocket |
|---|---|---|
| 接続 | リクエストごとに開く・閉じる | 一度開いたら維持する |
| 通信の向き | クライアント → サーバ（要求）→ レスポンス | 双方向に自由 |
| サーバ起点の通知 | 苦手（ポーリングが必要） | 得意（いつでも送れる） |
| 用途の例 | ウェブページの取得、API呼び出し | チャット、ゲーム、リアルタイム通知 |

## 本プロジェクトでの使われ方

本プロジェクトでは、ゲームサーバとエージェントの間の通信に WebSocket を使っています。
ゲーム中、サーバはエージェントに対して「いま発言してください」「投票先を教えてください」といったリクエストを **何度も** 送ってきます。HTTP のように毎回つなぎ直すと無駄が多いので、WebSocket で **接続を維持したまま** やり取りを続けます。

URLはふつうの `http://` ではなく `ws://`（暗号化する場合は `wss://`）から始まります。
ローカル起動の既定値は次の通りです。

```
ws://127.0.0.1:8080/ws
```

このURLの意味を簡単に分解すると次のようになります。

| 部分 | 意味 |
|---|---|
| `ws://` | WebSocket で通信するという目印（HTTPの `http://` と同じ役割） |
| `127.0.0.1` | **自分自身のPC** を指す特別なIPアドレス（`localhost` と同じ）。同じPCの中でサーバとエージェントを動かすときに使う |
| `8080` | **ポート番号**。1台のPCの中で「どのプログラム宛か」を区別する番号で、aiwolf-nlp-server は既定で 8080 番を使う |
| `/ws` | サーバ内のどの入口（エンドポイント）に接続するかを示すパス |

> サーバを別のPCで動かす場合は `127.0.0.1` の部分をそのPCのIPアドレスやホスト名に置き換えます。ポート番号もサーバの設定で変更可能です。

メッセージの中身は **JSON文字列** で、サーバが送ってくるリクエストの種類（`TALK` `VOTE` など）や現在のゲーム状況が含まれます。エージェントはこれを受け取り、必要に応じて返答を JSON で返します。

> 詳しいリクエストの種類や返答の流れは [エージェントとサーバの仕組み](../overview/system_mechanism.md) を参照してください。
