## ターミナルとは？

**ターミナル（Terminal）** は「文字で命令を打ち込む」ための画面です。  
マウスで操作する Windows の **GUI（グラフィカルユーザーインターフェイス）** と違い、**コマンドを入力して直接コンピュータに指示**を出します。

| 操作方法 | 例 | 特徴 |
|---|---|---|
| GUI（グラフィカル） | フォルダを開く、右クリックでコピー | 直感的だが、操作が遅く手順が多い |
| CLI（コマンドライン） | `cp file.txt backup/` | コマンドを覚える必要があるが、高速で自動化しやすい |

> WSL で開く **黒い画面（bash）** がターミナルです。最初は戸惑いますが、**GUIでできることを文字で正確に再現**できる強力なツールです。

---

## WSLとは何か

**WSL（Windows Subsystem for Linux）** は、Windows 上で Linux を動かす仕組みです。

```text
[Windows カーネル]
    └─ WSL 互換レイヤー（Linux の命令を Windows に橋渡し）
         └─ Linux ユーザー空間（bash, コマンド群, ファイル階層）
````

* **WSL2** が標準。軽量な仮想化で Linux カーネルを動かします。
* Linux のコマンドやパッケージ管理（`apt` など）が **そのまま使える**のが利点。

---

## Linuxの設計思想

* **小さい道具を組み合わせる**（Unix哲学）
  1 個のコマンドは 1 つの役割。**パイプ**で連結して機能を足す。
* **入出力は共通の仕組み**（標準入出力 / ファイルI/O）
  「読む / 書く」を同じ考え方で扱うため、学習がシンプル。

## WSL のインストールと起動

### 🔹 1. インストール（Windows 10/11）

PowerShell またはターミナルを**管理者として**開いて次を実行：

```powershell
wsl --install
```

これで自動的に **WSL2 + Ubuntu** が導入されます（再起動が必要な場合あり）。

### 🔹 2. 初回起動とユーザー設定

インストール後、Windows ボタン → **「wsl」** または **「Ubuntu」** と入力して起動。
初回は **ユーザー名／パスワード** を設定します。

以降の起動方法：

* スタート → **Ubuntu** または **Windows Terminal**
* PowerShell から `wsl`

## WSLの基本

例：

```bash
alice@WIN11PC:/home/alice/app (main) $
```

* **ユーザー名**：`alice` … 実行しているユーザー
* **ホスト名**：`WIN11PC` … Windows側マシン名
* **カレントディレクトリ**：`/home/alice/app` … 今いる場所（`~`は自分のホームの短縮）
* **状態表示**：`(main)` … Gitブランチなど（テーマ次第）
* **プロンプト記号**：`$` … 入力待ち（管理者は`#`）

### 初期位置とルート移動（WSL）

* **WSLを開くとだいたい**：`~`（= `/home/<ユーザー名>`）にいる
* **ルートへ移動**：`cd /`
* **ホームへ戻る**：`cd` だけ、または `cd ~`
* **Windows側のCドライブ**：`/mnt/c/...`

### 基本の流れ

* **通常**：コマンドを実行 → 終了 → **プロンプトが戻る**。

### ただしプロンプトが返ってこないパターンもあります

1. **フォアグラウンド実行**：サーバーなどが前面で動き続けている
2. **インタラクティブREPL**：`python` や `node` 等の対話モードに入っている
3. **フルスクリーン系**：`vim` / `nano` / `htop` など画面占有アプリを開いている
4. **ページャ**：`man` や `git log` で `less` が起動してスクロール待ち
5. **入力待ち**：パスワードや `(y/N)` の確認待ちでブロックしている
6. **継続出力の“追い”**：`tail -f` / `journalctl -f` などを見続けている

（いずれも異常ではなく、終了や離脱操作をすればプロンプトは戻ります。例：`Ctrl-C` で中断、`q` や `:q` で終了、REPLは `exit()` / `Ctrl-D`、エディタは保存終了など。）
必用であれば新しくターミナルを起動してください。

## コマンドのオプション（-a, -l など）

Linux のコマンドは、**基本形（動詞）＋オプション（形容詞）** の構造です。

```bash
ls -l -a
```

* `ls` … **list**（一覧表示）
* `-l` … **long** format（詳細表示）
* `-a` … **all**（隠しファイルも表示）

つまり **「長く詳しく全部表示」** の意味。

| 形式          | 意味               | 例           |
| ----------- | ---------------- | ----------- |
| `-a`        | 短いオプション（1文字）     | `ls -a`     |
| `--all`     | 長いオプション（読みやすい）   | `ls --all`  |
| `--help`    | 使い方を表示           | `ls --help` |
| `man コマンド名` | manual（マニュアル）を開く | `man ls`    |

**📝 コツ**
英単語の頭文字が多い（`-r`=reverse, `-f`=force, `-v`=verbose など）。
**意味を暗記**するより、**略語を読む**感覚で理解すると楽です。

## 標準入出力とリダイレクト・パイプ

| 記号     | 作用（最低限だけ覚える）                    | 例                         |
| ------ | ------------------------------- | ------------------------- |
| `\|`   | 左コマンドの**標準出力** → 右コマンドの**標準入力** | `cat a.txt \| grep hello` |
| `>`    | **標準出力**をファイルに**上書き**           | `echo hi > out.txt`       |
| `>>`   | **標準出力**をファイルに**追記**            | `echo hi >> out.txt`      |
| `<`    | ファイルを**標準入力として読む**              | `sort < data.txt`         |
| `2>`   | **標準エラー出力**をファイルへ               | `mycmd 2> err.log`        |
| `2>&1` | **エラー出力を標準出力に合流**               | `mycmd > all.log 2>&1`    |

### 組み合わせ例

```bash
# エラーを含む全出力を保存
python app.py > run.log 2>&1

# ログから ERROR を数えて多い順に表示
grep -h "ERROR" *.log | sort | uniq -c | sort -nr | head

# 行頭だけ抜き出し（; 区切りの1列目）→ ユニーク数
cut -d';' -f1 input.csv | sort | uniq -c | sort -nr
```
