このページでは、本番の大会サーバにエージェントを接続する方法を説明します。
予選・本戦ともに、基本的な流れは **「配布された接続情報を `config.yml` に書いてエージェントを起動するだけ」** です。

---

## 事前に配布される情報

参加登録が済むと、運営から **Slackなど** を通じて次の情報が案内されます。

* **接続先URL**（例：`wss://example.com/ws`）
* **認証トークン**（大会参加者ごとに配布されます）
* **ゲーム形式と人数**（5人戦・9人戦・13人戦のいずれか）
* **対戦期間とスケジュール**

> 配布される情報は大会ごとに変わります。最新の案内は必ず運営からの連絡を確認してください。

---

## `config.yml` の設定例

`config/config.yml` の `web_socket` セクションを、以下のように書き換えます。

```yaml
web_socket:
  url: wss://example.com/ws   # 運営から配布された接続先URL
  token: abcdef1234567890     # 運営から配布されたトークン
  auto_reconnect: true        # 本戦では true を推奨

agent:
  num: 5                      # 対戦形式に合わせる（5 / 9 / 13）
  team: kanolab               # 参加登録時のチーム名に合わせる
  kill_on_timeout: true
```

**ここがポイント**：

* **`auto_reconnect: true`** にしておくと、試合終了後に自動で次の試合に接続できます。本戦では特に重要です。
* **`agent.num`** は必ず運営が指定する人数に合わせてください。
* **`agent.team`** は参加登録したチーム名と一致させてください。自己対戦モードでのマッチングやログに使われます。

---

## エージェントの起動

通常のローカル対戦と同じコマンドで起動します。

```bash
cd ~/aiwolfdial/aiwolf-nlp-agent-llm
uv run python src/main.py
```

起動後はそのまま放置しておけば、対戦開始の時間になると自動的にゲームが始まります。

## よくある疑問・注意点

### Q. 接続エラーが出続ける

`[Errno 61] Connection refused` のようなエラーが出る場合、次のいずれかが原因です。

* 接続先URL（`web_socket.url`）が間違っている
* 認証トークン（`web_socket.token`）が間違っている
* 運営サーバがメンテナンス中

メンテナンス中の場合、`auto_reconnect: true` にしておけば、サーバが復旧したときに自動でつながります。

### Q. 試合の合間に設定を変えたい

試合の途中で設定を変更したい場合は、一度エージェントを停止して `config.yml` を書き換え、再度起動してください。
運営サーバでは **同じチーム名の別エージェント** は別々にマッチングされるので、複数バージョンを同時に走らせることもできます。

### Q. ログはどこに残る？

サーバ側のログは運営が保持しますが、エージェント側のログは通常通り `aiwolf-nlp-agent-llm/log/` 以下に保存されます。
対戦後に自分の挙動を確認したいときは、ここを見てください。

## 本番前のチェックリスト

* [ ] 接続先URLとトークンを正しく設定した
* [ ] `agent.num` がサーバの人数と一致している
* [ ] `auto_reconnect: true` にしてある（本戦の場合）
* [ ] APIキーが `.env` に入っていて、残量に余裕がある
* [ ] ローカルで一度通しで動かして、ログが正しく出ることを確認した
