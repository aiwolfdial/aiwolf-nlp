## 仮想環境ツールの比較

| ツール          | 概要                       | 得意なこと（長所）                             | 苦手なこと（注意点）                                 | ロック/再現性                          | 非Python依存 | 速度/UX       | 典型用途                     |
| ------------ | ------------------------ | ------------------------------------- | ------------------------------------------ | -------------------------------- | --------- | ----------- | ------------------------ |
| **venv（標準）** | Python標準の軽量仮想環境          | 学習コスト最小・標準・軽量・どこでも動く                  | 依存解決/ロック機能は無し（`pip`と`requirements.txt`で補完） | `requirements.txt`で再現            | ✗         | 速い/シンプル     | 小〜中規模アプリ、簡潔な研究開発、本プロジェクト |
| **conda**    | Python以外も含む“環境”管理        | C系/科学系ライブラリ、GPU/MLスタックの一括解決           | Python以外も抱えるため重い・配布系が独自                    | `environment.yml`                | ◯（得意）     | 中（構築は重いが便利） | データサイエンス、GPU/ネイティブ依存     |
| **pipenv**   | pip＋venv＋Pipfile         | 依存解決/ロックを自動で管理、`.env`連携が簡便            | 開発停滞期があった・大規模で遅いことあり                       | `Pipfile.lock`                   | ✗         | 中（遅い場面あり）   | Webアプリ中規模、チームでlock前提     |
| **poetry**   | 依存解決＆ビルド/配布まで統合          | 依存解決が強い・`pyproject.toml`でモダン・パッケージ配布も | 設定が増える・学習コスト                               | `poetry.lock`                    | ✗         | 中（解決はやや重）   | ライブラリ開発、製品開発での厳密管理       |
| **uv**       | 超高速の pip/venv 代替（Rust実装） | インストール/解決が速い・一発で環境作成〜導入               | 新しめ・社内制約環境では導入可否要確認                        | `requirements.txt`/`uv.lock`（任意） | ✗         | **速い**      | 高速反復開発、CI/CD、PoC         |

> 迷ったら：**標準・軽量・癖が少ない venv** が無難。
> GPUやC依存が濃い科学計算では **conda**、依存厳密/配布まで見るなら **poetry**、高速試行は **uv** が候補です。

## venv マニュアル

### 作成

```bash
# プロジェクトのルートで
python -m venv .venv
```

* `.venv` は各IDE/ツールが検出しやすく、`.gitignore` にも入れやすい

### 有効化（Activate）

  ```bash
  source .venv/bin/activate
  ```

> プロンプトに `(.venv)` と出れば有効化成功。

### 無効化（Deactivate）

```bash
deactivate
```

### 依存の固定と配布

```bash
# いま入っているバージョンを固定
pip freeze > requirements.txt
```

### `.gitignore` に追加

```text
.venv/
```

> 仮想環境フォルダは **コミットしない** のが原則。

以上で仮想環境の説明を完了します。
