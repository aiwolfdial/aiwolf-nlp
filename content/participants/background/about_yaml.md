> このページは「YAML の文法そのもの」に絞って解説します。

## YAML って何？

**YAML**（ヤムル, *YAML Ain't Markup Language*）は、**人が読みやすい形式で構造化データを書くための文法** です。
役割としては JSON とほぼ同じで、辞書（キーと値）・リスト・文字列・数値・真偽値を表現できます。

JSON と比べると、

* 波カッコ `{}` や角カッコ `[]` を **書かなくていい**
* カンマ `,` も **省略できる**
* 代わりに **インデント（字下げ）で構造を表す**

という違いがあります。

たとえば次の JSON

```json
{
  "agent": {
    "num": 5,
    "team": "kanolab"
  },
  "llm": {
    "type": "google"
  }
}
```

は、YAML だと次のように書けます。

```yaml
agent:
  num: 5
  team: kanolab
llm:
  type: google
```

設定ファイルとしては YAML のほうが見通しがいいので、本プロジェクトでも採用されています。

---

## 4種類の値

YAML で覚えるべきデータの形は、基本的に4種類です。

| 形 | 例 | Python に直すと |
|---|---|---|
| **スカラー（単一値）** | `42` / `"ミナト"` / `true` | `int` / `str` / `bool` |
| **辞書（マッピング）** | `key: value` の集まり | `dict` |
| **リスト（シーケンス）** | `-` で始まる行の集まり | `list` |
| **null** | `null` / `~` / 空欄 | `None` |

---

## 辞書（キーと値）

`キー: 値` の形で書きます。コロンの **後ろには必ず半角スペース** が必要です。

```yaml
agent:
  num: 5
  team: kanolab
  kill_on_timeout: true
```

ネストするときは **インデントを下げる** だけで、上位の値の中に入ったことを表します。
インデントは半角スペース2つが慣例（タブは使わない、後述）。

---

## リスト

行頭に `-` を書くとリストの要素になります。

```yaml
roles:
  - WEREWOLF
  - SEER
  - VILLAGER
```

辞書のリストもよく出てきます。

```yaml
players:
  - name: ミナト
    role: SEER
  - name: サクラ
    role: VILLAGER
```

`{}` / `[]` を使ったインライン記法も書けますが、本プロジェクトでは **ブロック形式（インデント）** が使われています。

---

## 文字列のクォート

YAML の文字列は、**基本的にクォートなしでそのまま書けます**。

```yaml
team: kanolab
agent_name: ミナト
```

ただし、次のようなときは **明示的にクォートで囲む** 必要があります。

| 状況 | 書き方 |
|---|---|
| `:` や `#` を文字として含めたい | `description: "12:00 から開始"` |
| 先頭に `-` や `?` がある | `text: "- これは項目ではない"` |
| 文字列を `true` / `false` / `yes` / `no` / 数値と区別したい | `flag_str: "true"` |

ダブルクォート `"..."` ではエスケープシーケンス（`\n` など）が使え、シングルクォート `'...'` ではエスケープが効かない（リテラルとして扱う）違いがあります。

---

## ブロックスカラー（複数行の文字列）

本プロジェクトの `config.yml` で **いちばん登場するパターン** が、`prompt.*` のような **複数行の文字列** です。
これには `|` や `>` といった専用の記号を使います。

| 記号 | 改行の扱い | 末尾の改行 |
|---|---|---|
| `\|` | **そのまま保持**（改行は改行のまま残る） | 1つだけ残す |
| `\|-` | そのまま保持 | **削除** |
| `\|+` | そのまま保持 | すべて残す |
| `>` | **空白に折りたたむ**（連続行は1つの段落として連結） | 1つだけ残す |
| `>-` | 折りたたむ | 削除 |

本プロジェクトの `prompt.*` は、ほぼ全部 `|-` です。

```yaml
prompt:
  talk: |-
    あなたは{{ info.agent }}です。
    {{ info.day }}日目の議論中です。
    自然な日本語で1発言してください。
```

`|-` の **直後の改行は無視** され、ブロックの **行頭インデント（半角スペース）** はそろえれば解釈時に取り除かれます。
つまり、上の例では LLM に渡る文字列は「あなたは…」「{{ info.day }}…」「自然な…」の3行になります。

> **ポイント**：`prompt.talk: |-` の下にあるテンプレート行の行頭インデントは、**全行で同じ深さ** にそろえてください。
> 一行だけ深く／浅くなっていると、その行が「他より深い／浅い」と解釈されて中身が崩れます。

---

## 真偽値・null・数値

```yaml
auto_reconnect: false   # bool
sleep_time: 3           # int
temperature: 0.7        # float
token: ~                # null（"" や省略でも null になる）
```

YAML 1.1 系では `yes` / `no` / `on` / `off` も真偽値として解釈される実装があります。
**「文字列のつもりで書いた値が bool に化ける」事故** を避けるため、文字列として扱いたいときは必ずクォートしてください（例：`token: "yes"`）。

---

## コメント

`#` 以降は行末までコメントとして無視されます。

```yaml
agent:
  num: 5      # 同時起動するエージェント数
  team: kanolab
```

ブロックコメント（複数行を一気に）はないので、行ごとに `#` を入れてください。

---

## インデントの罠

YAML で **9割のエラーはインデント由来** です。次の3つだけ守れば、ほぼ事故りません。

### 罠1：タブを使わない

YAML は **タブ文字を文法エラー** として扱います。エディタの設定で「Tab を半角スペースに変換」をオンにしておくと安全です。

### 罠2：階層ごとにインデント幅をそろえる

同じ階層の項目は **同じ深さ** に書きます。

```yaml
# OK
agent:
  num: 5
  team: kanolab

# NG（num と team の階層が揃っていない）
agent:
  num: 5
   team: kanolab
```

### 罠3：全角スペースを混ぜない

見た目は半角と区別しづらいですが、YAML パーサは別物として扱います。
コピー＆ペーストで紛れ込みやすいので、エラーが出たら **見えない文字** を疑ってください。

## 本プロジェクトでの使われ方

本プロジェクトでは、エージェント側の `config/config.yml` がすべての設定の入口になっています。

```yaml
agent:
  num: 5
  team: kanolab

llm:
  type: google

prompt:
  talk: |-
    あなたは{{ info.agent }}です。{{ info.day }}日目です。
    最新の会話:
    {% for w in talk_history[sent_talk_count:] -%}
    {{ w.agent }}: {{ w.text }}
    {% endfor %}
```

これは `main.py` の起動時に Python の辞書として読み込まれ、エージェント内では `self.config["agent"]["num"]` のような形で参照されます。
つまり、**YAML を編集することは、Python の辞書を編集するのと同じ意味** です。

`prompt.*` の中身が **Jinja2 テンプレート** であることだけ別の話なので、そちらは [Jinja2テンプレートとは](./about_jinja2.md) を参照してください。

## よくあるエラーと対処

| 症状 | 主な原因 |
|---|---|
| `yaml.scanner.ScannerError: mapping values are not allowed here` | コロンの後ろに半角スペースがない／文字列内のコロンをクォートしていない |
| `yaml.scanner.ScannerError: found character '\t'` | インデントにタブが混入している |
| `yaml.parser.ParserError: ... could not find expected ':'` | インデント幅がそろっていない／全角スペースが混ざっている |
| 値が文字列のつもりだったのに bool になる | `true` / `false` / `yes` / `no` などをクォートする |
| `\|-` ブロックの中身が想定と違う | ブロック内の行頭インデントが揃っていない |

---

## まとめ

* YAML は「インデントで構造を表す」設定言語。JSON とほぼ同じことが書ける
* 辞書は `key: value`、リストは `- item`、複数行文字列は `\|-` がよく使われる
* タブ・全角スペース・インデント不揃いの3つで事故りやすい
* `prompt.*` のような複数行文字列は、Jinja2 テンプレートとして処理される
