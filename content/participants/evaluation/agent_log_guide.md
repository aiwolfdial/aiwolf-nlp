このページでは、エージェント側のログの読み方を説明します。
エージェントログには **「LLMが何を入力され、何を返したか」** が記録されていて、改良のヒントを探すときに最も役立つ情報源です。

---

## ログはどこに保存される？

サンプルエージェントを動かすと、`aiwolf-nlp-agent-llm/log/` 配下にログファイルが作られます。

```text
aiwolf-nlp-agent-llm/
└── log/
    └── 20251012233100123/    # タイムスタンプのフォルダ（1ゲームごと）
        ├── kanolab1.log       # エージェント1体のログ
        ├── kanolab2.log       # エージェント2体目
        └── ...
```

1ゲームあたり、エージェントの人数だけファイルが作られます（5人戦なら5ファイル）。

---

## 1行のフォーマット

各行は次のような形をしています。

```text
<タイムスタンプ> - <エージェント名> - <ログレベル> - <メッセージ>
```

* **ログレベル**：`DEBUG`（詳細）・`INFO`（重要）・`WARNING`（警告）・`ERROR`（エラー）
* **DEBUG行**：サーバから届いたパケット全体
* **INFO行**：LLMに送ったプロンプト、LLMからの返答、最終的にサーバに送った内容

> ログのレベルや出力先は `config.yml` の `log` セクションで変更できます。詳しくは [config.ymlの説明](../enhancement/config_explanation.md) をご覧ください。

---

## DEBUG行：受信パケットの中身

サーバから届いたパケットの生の内容が `Packet(...)` として記録されます。ここには次のような情報が入っています。

* `request`：リクエストの種類（`INITIALIZE` / `TALK` / `VOTE` など）
* `info`：現在のゲーム状態（日付・自分の役職・生死・占い結果など）
* `setting`：ゲームの設定（発言制限・タイムアウトなど）
* `talk_history` / `whisper_history`：新しく届いた会話の差分

「今、自分のエージェントに何が見えているのか」を確認したいとき、この行を眺めるのがいちばん確実です。

---

## INFO行：LLMのやり取りと最終返答

INFO行は2種類あります。

1. **LLMへの入出力**：`['LLM', プロンプト, LLMの応答]`\
   LLMに実際に送ったプロンプトと、LLMが返した文章が記録されます。
2. **サーバへの返答**：`['Request.XXX', 応答内容]`\
   LLMの応答をそのままサーバに送った内容です。発言・投票・占い先など、**最終的にゲームに反映された値** がここに残ります。

この2つを見比べると、「LLMがどう考えて、どんな行動につながったか」がわかります。
プロンプトを改良したいときは、まずここを読むところから始めましょう。

---

## ログの例

実際のログはこんな雰囲気です（見やすく抜粋しています）。

```text
2025-10-12 23:31:00,742 - kanolab2 - DEBUG - Packet(request=INITIALIZE, info=..., setting=...)
2025-10-12 23:31:04,793 - kanolab2 - INFO - ['LLM', 'あなたは人狼ゲームのエージェントです。…', '了解しました。ケンジです。よろしくね！']
2025-10-12 23:31:19,284 - kanolab2 - INFO - ['LLM', 'トークリクエスト\n履歴:\n…', 'ヴィクトリアさん、誰のことですか？']
2025-10-12 23:31:19,284 - kanolab2 - INFO - ['Request.TALK', 'ヴィクトリアさん、誰のことですか？']
```

このエージェントは「ケンジ」というキャラクターとしてゲームに参加し、昼の会話で「ヴィクトリアさん、誰のことですか？」と発言していることがわかります。

---

## 出力形式のカスタマイズ

ログの出力処理は `src/utils/agent_logger.py` に書かれています。
デバッグしやすいように書式を変えたり、特定の情報だけ抜き出したくなったら、このファイルを編集してください。

---

## デバッグのコツ

ログは手で開くよりも、**リアルタイムで追いかける** のが圧倒的に楽です。
Linux / macOS / WSL なら、次のコマンドで特定のエージェントのログを追跡できます。

```bash
tail -f ~/aiwolfdial/aiwolf-nlp-agent-llm/log/*/kanolab1.log
```

うまくいかないときは、まず `config.yml` の `log.request.talk: true` にして、**実際にサーバに送った発言** を全部残すようにすると原因が追いやすくなります。
