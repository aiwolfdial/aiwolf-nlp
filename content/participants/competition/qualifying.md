こちらのページでは、大会の **予選フェーズ** での具体的な手順を説明します。

予選は **自分のエージェント同士で自己対戦** し、その **対戦ログを運営に提出** するフェーズです。本戦に進むためには、所定の期日までにこの提出を済ませる必要があります。

> 予選・本戦に共通する `config.yml` の書き方や、トラックの考え方は [大会の進め方（全体像）](./competition.md) を先にお読みください。

---

## 1. 予選で何が起きるか

予選フェーズでは、**運営が用意した予選サーバ** に接続するだけで、自動的に自己対戦が進行します。

```text
あなたのエージェント（5体 or 9体 or 13体）
       │
       │ WebSocket
       ▼
   予選サーバ（運営が用意・トラックごとに別ポート）
       │
       └─→ 自己対戦を 5回程度連続実行
              │
              ▼
          サーバ側にログが保存される（参加者がアクセスできるURL）
```

* エージェントは **自分のチームのエージェント同士で** 試合をします（他チームとは対戦しません）
* 1回の接続で **5試合程度** が連続実行されます（`auto_reconnect: true` 必須）
* 各試合のログは **サーバ側に自動保存** され、後で参加者が一覧から自分のものを取り出します

---

## 2. 必要な情報（運営から配布）

予選にあたって、運営から **Slack 等を通じて以下の情報** が案内されます。

* **トラックごとの予選サーバ接続先**（例：5人村用・9人村用などで別ポート）
* **大会設定ファイル**（試合数・発言文字数・スキップ可否などが書かれている。閲覧用）
* **ログ出力先 URL**（試合後に自分のログを探す場所）
* **ログファイルの提出方法**（Slack の該当スレッドに返信、など）

---

## 3. 設定（`config.yml`）

参加するトラックの数だけ、`config/config.yml` の `web_socket` と `agent` を書き換えます。
**[大会の進め方 §4](./competition.md#4-全フェーズに共通する-configyml-の設定)** で示したテンプレートと同じ要領です。

```yaml
web_socket:
  url: ws://（運営配布の予選用URL）/ws
  token:                # 配布された場合のみ設定
  auto_reconnect: true  # 連続で5試合まわす場合は true、1試合ずつ確認したい場合は false

agent:
  num: 5                # トラックの人数に合わせる
  team: kanolab         # 参加登録時のチーム名
  kill_on_timeout: true
```

> **`auto_reconnect` の使い分け**：予選では `true`／`false` のどちらでも構いません。
> `true` にすると 1 回の起動で **連続5試合をまとめて消化** でき、`false` にすると **1試合ごとに切断** されるので、挙動を確認しながら少しずつ進めたいときに便利です。

> **接続先はトラックごとに違います**：5人村と9人村で別ポート／別パスのことが多いので、混同に注意してください。

---

## 4. 実行と試合進行

通常のローカル対戦と同じコマンドで起動するだけです。

```bash
cd ~/aiwolfdial/aiwolf-nlp-agent-llm
uv run python src/main.py
```

起動すると、

1. エージェントが運営の予選サーバに接続
2. サーバが自動的に自己対戦を開始
3. 連続で5試合程度が進行（数分〜数十分かかることがあります）
4. すべての試合が終わると、サーバ側で次の接続待ちに戻ります

`auto_reconnect: true` の場合、サーバ側の都合（メンテナンス、再起動など）にも自動で追随します。

> **複数トラックに参加するとき**：トラックごとに別ターミナルでエージェントを起動してください。
> [大会の進め方 §5](./competition.md#5-起動方法) を参照。

---

## 5. ログを確認する

試合が終わったら、運営が案内している **ログ出力先 URL** にアクセスします。
そこには、参加チームごとの **`.log` ファイル** が並んでいます。

ファイル名はチーム名と試合内容を表す形式（例：`<タイムスタンプ>_<エージェント名>_..._.log`）で並び、自分のチームのものを見分けやすくなっています。

> **`.log` 形式のファイル** を提出します。同じ場所に `.json` 形式のファイルもありますが、提出に使うのは `.log` 形式の方です。

---

## 6. ログを提出する

提出方法は大会ごとに案内されますが、**「ログ出力先から提出するファイル名をコピーして、Slack の該当スレッドに返信する」** という流れが定番です。

```text
（提出例）
133.167.32.100/aiwolf/.../log/<提出するログファイル名>.log
```

> **提出時の注意**：
>
> * **ファイル本体ではなく URL／ファイル名** を提出するスタイルが一般的です（運営側でログを保管しているため）
> * `.log` 形式を指定（`.json` ではなく）
> * 1つのトラックにつき **1つのログファイル** を提出する形が多い（運営の案内に従ってください）
> * 提出締切は **本戦開始時刻の直前** に設定されます。**遅れると自動辞退** とみなされる場合があります

---

## 7. 期日までにやること

予選フェーズで参加者がやるべきことを時系列で並べると、次のようになります。

1. 運営の案内を確認し、**トラックごとの接続先・設定** を把握
2. `config/config.yml` を **トラックごとに用意**（コピーしておくと楽）
3. APIキーの残量を確認・必要なら課金プラン引き上げ
4. 各トラックでエージェントを起動し、**自己対戦が完走するまで待機**
5. ログ出力先から **自分のログファイル名** を確認
6. 期日までに **Slack スレッド等で提出**

---

## 8. よくあるトラブル

### 接続できない（`Connection refused` など）

* `web_socket.url` が予選用 URL になっているか（ローカル対戦のままになっていないか）
* `web_socket.auto_reconnect: true` か（`false` だと一度切れたら戻らない）
* 起動時刻が運営公開時間より前ではないか

### 試合が始まらない／途中で止まる

* `agent.num` が **トラックの人数** と一致しているか
* `agent.team` を **参加登録時のチーム名** に合わせているか
* APIキーのレート制限・残量を確認

### ログが出力先に見つからない

* 試合が完走していない可能性（タイムアウトで全滅など）
* 同名ファイルが複数あって混乱している場合は、**タイムスタンプ最新** を選ぶ
* 不明な場合は運営に相談

---

## 9. 提出が完了したら

予選提出が完了したら、続いて [本戦編](./main_round.md) を確認しておきましょう。

本戦は **開始時刻までにエージェントを起動・待機状態** にしておく必要があります。準備期間が短い場合があるので、提出後すぐに目を通しておくと安心です。
