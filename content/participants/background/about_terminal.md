このページは、[WSLとLinux](./about_wsl_linux.md) で用意した環境を **実際に操作する** ための実用ページです。
前半でターミナル操作の土台になる考え方（プロンプト・パス）を押さえ、後半に開発で最低限使うコマンドをまとめています。

---

## ターミナルとは

**ターミナル（Terminal）** は、「文字で命令を打ち込んで」コンピュータを操作するための画面です。
マウスでフォルダを開いたりコピーしたりする Windows の操作（**GUI**）に対して、ターミナルは **文字で直接指示を出す** やり方です。

| 操作方法 | 例 | 特徴 |
|---|---|---|
| GUI（マウス操作） | フォルダを開く、右クリックでコピー | 直感的だが、手順が多い |
| ターミナル（文字操作） | `cp file.txt backup/` | 覚える必要はあるが、速く、自動化しやすい |

WSL を起動したときに開く **黒い画面** が、このターミナルです。

---

## コマンドとは

**コマンド** は、ターミナルに打ち込む「命令の言葉」です。たとえば `ls` は「ファイル一覧を表示せよ」という命令です。
ターミナルにコマンドを入力して **Enter** を押すと、その命令が実行されます。

Linux のコマンドの多くは **英単語やその略語** になっており、何の略かが分かると意味を覚えやすくなります（`ls` = list、`cp` = copy など）。代表的なものは、このページの後半にまとめています。

---

## プロンプトとは

ターミナルでコマンドを打つ前に、必ず確認したいものがあります。**プロンプト** です。

**プロンプトは「入力待ちの行」** です。ターミナルがコマンドを受け付けられる状態のときに表示されます。

```bash
ユーザー名@PC名:~$
```

逆に言うと、**プロンプトが表示されていないときは、コマンドを入力してはいけません**。前のコマンドがまだ動いている最中であり、その状態で打ち込んでも正しく実行されないためです。「プロンプトが出ている＝入力してよい合図」と覚えてください。

---

## プロンプトの読み方

プロンプトには、「今どこで・誰として操作しているか」の情報が詰まっています。

```bash
alice@WIN11PC:/home/alice/app (main) $
```

* **ユーザー名**：`alice` … 操作しているユーザー
* **ホスト名**：`WIN11PC` … マシン名
* **現在地**：`/home/alice/app` … 今いるフォルダ（`~` は自分のホームの短縮表記）
* **状態表示**：`(main)` … Git のブランチなど（環境による）
* **入力記号**：`$` … 入力待ち（管理者のときは `#`）

### プロンプトが返ってこないとき

コマンドは普通、実行 → 終了 → **プロンプトが戻る**、と進みます。
ですが、次のような場合はプロンプトが戻らず、入力待ちに見えないことがあります。いずれも異常ではありません。

1. **サーバなどが動き続けている**（前面で実行中）
2. **対話モードに入っている**（`python` や `node` を起動した状態）
3. **画面を占有するアプリ**（`vim` / `nano` / `htop` など）
4. **ページャでスクロール待ち**（`man` や `git log` で `less` が開いた状態）
5. **入力待ちでブロック**（パスワードや `(y/N)` の確認待ち）
6. **出力を見続けている**（`tail -f` などで追従中）

抜けるには、`Ctrl-C` で中断、`q` や `:q` で終了、対話モードは `exit()` / `Ctrl-D`、エディタは保存して終了、などの操作をします。
**迷ったら、新しくターミナルを開き直すのが確実です**（Windows ボタン → `wsl` → Enter）。

## 絶対パスと相対パス

コマンドでファイルやフォルダを指定するとき、その「場所（パス）」の表し方には2通りあります。ここが分かると、コマンド操作がぐっと楽になります。

* **絶対パス** … **ルート（一番上）から** 目的のフォルダ／ファイルまでの道のり。
* **相対パス** … **今いる場所（現在地）から** 目的のフォルダ／ファイルまでの道のり。

道のりを表すのに、次の記号を使います。

* **`/`** … ルート（一番上の階層）。絶対パスはここから始まる。
* **`.`** … 現在地（今いるフォルダ）。
* **`..`** … 1つ上の階層。

たとえば WSL に入った直後（現在地が `/home/ユーザー名`）に、`sentan` というフォルダを作ったとします。この `sentan` フォルダの場所は、

* **絶対パス**：`/home/ユーザー名/sentan`（ルートから全部たどる）
* **相対パス**：`./sentan`（今いる場所から見て、すぐ下）

と、2通りで表せます。**どちらも同じ場所を指していて、起点が違うだけ** です。

> 移動コマンド `cd` でも同じです。`cd /`（絶対パスでルートへ）、`cd ..`（相対パスで1つ上へ）、`cd ~`（ホームへ）のように使い分けます。今いる場所は `pwd` で確認できます。

## 代表的なコマンド

コマンドは、後ろに **対象（パス）** をいくつ取るかで形が決まります。まずはこの3つの型を押さえると、初めて見るコマンドも読めるようになります。

### 型1：対象を1つ取る（`コマンド 対象`）

「どこ／どれに対して」を1つ指定する形です。

```bash
cd ./sentan        # sentan フォルダへ移動
ls sentan          # sentan の中を一覧表示
cat ./config.yml   # config.yml の中身を表示
code .             # 今いるフォルダを VS Code で開く
```

| コマンド | 由来 / 意味 | 用途 |
|---|---|---|
| `cd` | **change directory** | 指定したフォルダへ移動 |
| `ls` | **list** | 指定した場所の中身を一覧表示（省略すると現在地） |
| `cat` | **concatenate** | ファイルの中身を表示 |
| `grep` | **global regular expression print** | ファイルの中から文字列を検索 |
| `code` | **（VS Code）** | 指定したフォルダ／ファイルを VS Code で開く（`code .` で現在地を開く） |

### 型2：対象を2つ取る（`コマンド 元 先`）

「元」と「先」の2つを指定する形です。**順番が決まっている**（左が元、右が先）ので注意します。

```bash
cp old.txt new.txt        # old.txt を new.txt という名でコピー
mv old.txt sentan/        # old.txt を sentan フォルダへ移動
mv old.txt new.txt        # （同じ階層への移動なら）名前の変更になる
```

| コマンド | 由来 / 意味 | 用途 |
|---|---|---|
| `cp` | **copy** | ファイルをコピー（元 → 先） |
| `mv` | **move** | ファイルを移動／名前変更（元 → 先） |

### 型3：そのほか（対象なし・新規作成・削除）

| コマンド | 由来 / 意味 | 用途 |
|---|---|---|
| `pwd` | **print working directory** | 今いる場所を表示（対象は取らない） |
| `mkdir` | **make directory** | 新しいフォルダを作る（`mkdir sentan`） |
| `touch` | **touch** | 新しい空ファイルを作る（`touch memo.txt`） |
| `rm` | **remove** | ファイルを削除（`rm memo.txt`） |
| `sudo` | **superuser do** | **後ろに別のコマンドを取り**、それを管理者権限で実行（`sudo apt update`） |

> **`rm` は要注意**：削除はゴミ箱を経由せず、**すぐに消えて元に戻せません**。対象をよく確認してから実行してください（フォルダごと消す `rm -rf` は特に強力です。次節参照）。
> **`sudo` だけ毛色が違います**：ほかのコマンドが「対象（パス）」を取るのに対し、`sudo` は **後ろにコマンドそのもの** を取ります（`sudo （実行したいコマンド）`）。ソフトの導入など、管理者権限が必要なときに前に付けます。

### パッケージ管理・開発環境

ソフトの導入や Python の環境づくりで使うものです。詳しい使い方は各ツールのページや公式情報を参照してください。

| コマンド | 由来 / 意味 | 用途 |
|---|---|---|
| `apt` | **Advanced Package Tool** | パッケージ（ソフト）の導入・更新（例：`sudo apt install git`） |
| `pip` | **Pip Installs Packages** | Python パッケージの導入 |
| `python3 -m venv` | **virtual environment** | Python の仮想環境を作る |
| `source` | **source（読み込む）** | シェル設定や仮想環境を読み込む（例：`source .venv/bin/activate`） |

> ここに挙げたのは、開発を始めるのに最低限のコマンドだけです。ほかにも多くのコマンドがありますが、必要になったときに `man コマンド名`（次節）や検索で調べれば十分です。

## コマンドのオプション（-r, -a など）

**オプション** とは、コマンドの後ろに付けて **動作を細かく変える追加指示** です。同じコマンドでも、付けるオプションによって振る舞いが変わります。

形は2種類あります。

* **`-文字`**（短い形）… 1文字。例：`ls -a`
* **`--単語`**（長い形）… 単語で書くので読んで意味が分かる。例：`ls --all`

短い形は **続けて書けます**。`ls -l -a` は `ls -la` とまとめても同じです。

```bash
ls -l -a
```

* `ls` … **list**（一覧表示）
* `-l` … **long**（詳しく表示）
* `-a` … **all**（隠しファイルも表示）

→ つまり「**長く詳しく、全部表示**」という意味になります。

### 例：`rm -rf`（強力なので必ず意味を理解する）

削除コマンド `rm` によく付けるオプションです。

* `-r` … **recursive**（フォルダの中身ごと、まとめて）
* `-f` … **force**（確認を求めずに強制実行）

```bash
rm -rf sentan    # sentan フォルダを、中身ごと・確認なしで削除
```

これは「フォルダを中身まるごと問答無用で消す」命令です。**消したものは元に戻せません**。`.git`（→ Git の履歴）や `.venv`（→ 仮想環境）のような大事なフォルダに対して実行すると取り返しがつかないので、対象を声に出して確認するくらいの慎重さで使ってください。

### 使い方が分からないとき

オプションは英単語の頭文字が多く（`-r` = recursive、`-f` = force、`-v` = verbose など）、暗記するより **略語を読む** 感覚で十分です。確認したいときは次の2つが使えます。

| 方法 | 内容 | 例 |
|---|---|---|
| `--help` | コマンドの使い方を簡単に表示 | `ls --help` |
| `man コマンド名` | マニュアル（manual）を表示 | `man ls`（`q` で終了） |

## 隠しファイル（ドットファイル）

ファイル名やフォルダ名が **`.`（ドット）で始まる** ものは、Unix/Linux/macOS の慣習で **隠しファイル（ドットファイル）** と呼ばれ、通常の一覧には表示されません。「日常の操作では邪魔にならないが、必要なときだけ触る設定ファイル」を表すための仕組みです。

開発まわりの設定の多くがこれにあたります。

| 例 | 中身 |
|---|---|
| `.env` | APIキーなどの秘密情報（→[環境変数と.envファイル](./about_env_vars.md)） |
| `.gitignore` | Git で追跡しないファイルの指定 |
| `.git/` | リポジトリの履歴本体（ディレクトリ） |
| `.venv/` | Python の仮想環境（ディレクトリ） |
| `.bashrc` / `.profile` | シェル起動時に読まれる設定 |
| `.vscode/` | VS Code のプロジェクト設定（ディレクトリ） |

### コマンドでの見方・触り方

GUI では見えなくても、**コマンドからは普通に見えて操作できます**。隠しているのは「一覧での表示」だけで、アクセスが制限されているわけではありません。

```bash
ls               # 隠しファイルは出てこない
ls -a            # . で始まるものも含めて全部表示（a = all）
ls -la           # 詳細表示つきで全部（権限・サイズ・更新日時も）
nano config/.env # ファイルを開いて編集（nano は簡単なテキストエディタ）
```

> **`.` と `..` について**：`ls -a` すると `.`（現在地）と `..`（1つ上）も出てきます。これらは隠しファイルではなく、**全ディレクトリに必ず存在する特別な参照** です（絶対パス・相対パスで使ったあの `.` `..` です）。

### ドットで始まるディレクトリは不用意に消さない

`.git/` を消すと **Git の履歴がすべて失われ**、`.venv/` を消すと仮想環境が壊れます。`rm -rf .git` のような操作は取り返しがつかないので、ドットで始まるディレクトリを消すときは中身を理解してからにしてください。

> **Windows（エクスプローラ）で見たいとき**：「表示 → 隠しファイル」を有効にする必要があります。WSL を使っているなら、ターミナルから `ls -a` で確認するのが確実です。

## コマンドをつなぐ：パイプ `|`

`|`（パイプ）は、**左のコマンドの出力を、右のコマンドの入力として渡す** 記号です。

```bash
cat run.log | grep ERROR    # run.log の中身を grep に渡し、ERROR を含む行だけ表示
```

小さなコマンドを `|` でつないで、大きな処理を組み立てる——これは [WSLとLinux](./about_wsl_linux.md#補足linux-の設計思想なぜ開発で好まれるか) で触れた「小さな道具をつなぐ」という Linux の考え方そのものです。

> 出力をファイルに書き出す `>` や、エラー出力を扱う `2>` などの記号もありますが、必要になったときに調べれば十分です。
